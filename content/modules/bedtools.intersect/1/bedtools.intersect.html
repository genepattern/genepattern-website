Screen for overlaps between two sets of genomic features.

Author: Chet Birger, Broad Institute

Contact: gp-help@broadinstitute.org

Algorithm Version:

Introduction

BEDtools.intersect allows one to screen for overalaps between two sets of genomic features.  It allows one to have fine control as to how the intersections are reported.  The input files describing the two sets of genomic features may be BED/GFF/VCF or BAM files.

Parameters

Name Description
input file A  BED/GFF/VCF file A containing first set of genomic features. (-a)
bam input file A  BAM file A. Each BAM alignment in A is compared to B in search of overlaps. (-abam)
input file B  BED/GFF/VCF file B containing second set of genomic features. (-b)
unique  Report the presence of any (at least one) overlapping feature. (-u)
no overlap  Only report those entries in A that have no overlap in B. (-v)
count  Report the number of genes that each alignment overlaps. (-c)
original A  Report the orignal A feature. (-wa)
original B  Report the original B feature. (-wb)
min overlap  Minimum overlap required as a fraction of A. Default is 1E-9 (i.e. 1bp). (-f)
reciprocal  Require that the fraction of overlap be reciprocal for A and B. In other words, if -f is 0.90 and -r is used, this requires that B overlap at least 90% of A and that A also overlaps at least 90% of B. (-r)
bed output  When comparing alignments in BAM format (-abam) to features in BED format (-b), bedtools intersect will optionally write the output in BED format if set to true. (-bed)
header  Print the header from the A file prior to results. (-header)
uncompressed BAM  Write uncompressed BAM output. The default is write compressed BAM output. (-ubam)
left outer join  Perform a left outer join. (-loj)
write overlap size  Write size of overlap (in bases); choose between no, intersecting (-wo), all (-wao)
strandedness  Select strandedness constraints. (-s, -S)
split  Treat split BAM (i.e., having an N CIGAR operation) or BED12 entries as distinct BED intervals. (-split)
sorted  For very large B files, invoke a sweeping algorithm that requires position-sorted (e.g., sort -k1,1 -k2,2n for BED files) input. When using -sorted, memory usage remains low even for very large files. (-sorted)
output filename * Output filename

* - required

Input Files

  1. input file A
    The first of two BED-, GFF- or VCF-formatted files containing annotation track data.  BEDTools.intersect computes the overlap of features in input file A with the features in input file B.
  2. input file B
    The second of two BED-, GFF- or VCF-formated files containing annotation track data.  BEDTools.interesect computes the overlap of features in input file A with the features in input file B.

Output Files

  1. output filename
    See BEDTools intersect online documentation for a description of various output reporting formats.
