Introduction

The module takes a GCT and CLM file, and renames (and optionally reorders) the samples (ie the columns) in the GCT file. Optionally, a CLS file can be written using the information in the 3rd column of the CLM file. Note that all sample names in column 1 of the CLM file MUST be present in the GCT file. This module allows samples to be reordered and some samples to be excluded, since the order of the rows in the CLM file don’t have to match the order of the columns in the GCT file, and not all samples in the GCT file have to be mentioned in the CLM file.

Parameters

Name Description
gct file * GCT file
clm file * CLM file
gct out file * Name of the GCT file to be created
export cls * Export a CLS file?

* - required

Input Files

  1. gct.file

    This must be a valid GCT file. It will have sample names that you don’t like and want to change.

  2. clm.file

    This file maps the old sample names which are in your existing GCT file in column 1, 2 to the new names in column 2, and finally the class that each sample comes form in column 3.

  3. gct.out.file

    This is the name of the gct file that will be created.

  4. export.cls

    Do you want to export a cls file? yes or no.

Output Files

  1. A gct file with renamed sample names

The output file has the same number of samples, in the same order as specified in the CLM file. Note this allows you to reorder and rename the samples in one go. The number of genes is unchanged.
