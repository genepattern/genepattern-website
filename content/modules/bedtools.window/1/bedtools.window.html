Searches for overlapping features in A and B, adding a specified number (1000 by default) base pairs upstream and downstream of each feature in A.

Author: Chet Birger, Broad Institute

Contact: gp-help@broadinstitute.org

Algorithm Version:

Introduction

Similar to BEDTools.intersect, BEDTools.window searches for overlapping features in A and B; however, BEDTools.window adds a specifed number (1000, by default) of base pairs upstream and downstream of each feature in A.  This allows the detection of features in B that are "near" features in A.

Parameters

Name Description
input bed file A  Input BED file A. (-a)
input bam file A  Input BAM file A. (-abam)
input bed file B  Input BED/GFF/VCF file B. (-b)
base pairs added  Base pairs added upstream and downstream of each entry in A when searching for overlaps in B. Default is 1000 bp. (-w)
base pairs added upstream  Base pairs added upstream of each entry in A when searching for overlaps in B. Default is 1000 bp. (-l)
base pairs added downstream  Base pairs added downstream of each entry in A when searching for overlaps in B. Default is 1000 bp. (-r)
output filename * Output filename
additional command line options  Module will append these options to BEDTools window command line. See BEDTools window online documentation for supported options.

* - required

Input Files

  1. input bed file A
  2. input bed file B

Output Files

  1. Output filename
    When overlaps (extended by specified window) are found, both the original A feature and the original B feature are reported on the same line.
