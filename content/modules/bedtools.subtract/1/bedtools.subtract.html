Remove overlapping features.

Author: Chet Birger, Broad Institute

Contact: gp-help@broadinstitute.org

Algorithm Version:

Introduction

BEDTools.subtract searches for features in B that overlap A.  If an overalapping feature is found in B, the overalapping portion is removed from A and the remaining portion of A is reported.  if a feature in B overlaps all of a feature in A, the feature will not be reported.

BEDTools subtract online documentation

Parameters

Name Description
input file A * Input BED/GFF/VCF file A. (-a)
input file B * Input BED/GFF/VCF file B. (-b)
min overlap  Minimum overlap required as a fraction of A. Default is 1E-9 (i.e., 1bp)
strandedness  Strandedness contraint: none, same, different.  If enabled, only report hits in B that overlap A on the same/different strand. By default, overlaps are reported without respect to strand.
remove entire feature  Remove entire feature if any overlap: true, false.  Is false by default; i.e., only subtract portion of A that overlaps B.  When set to true, if any overlap is found (or min overlap) the entire feature is removed.
output filename * Output filename

* - required

Input Files

  1. input file A
    Input BED/GFF/VCF file A.  This is the BED file from which overlapping features are removed.
  2. input file B
    Input BED/GFF/VCF file B.  This is the BED file containing features for which overlaps in BED file A are searched for.

Output Files

  1. output filename
    BED/GFF/VCF file containing results of feature subtraction.
