Summary

The FlowFingerprinting module takes a flow set and generates a description of the multivariate probability distribution function of its flow cytometry data in the form of a fingerprint. As such, it is independent of a presumptive functional form for the distribution, in contrast with model-based methods such as Gaussian Mixture Modeling. Internally, this module uses the FlowFP package from R/BioConductor. FlowFingerprinting can be used for QC screening. 

For more information on the FCS file format, see the FCS 3.1 File Standard (PDF). Algorithm

Parameters

Name Description
Input FCS data files  The FCS file(s) to be fingerprinted. A zip or directory of FCS files.
Model FCS data file  Training data (FCS file) for the underlying model. If none, the input FCS data is used for creating a default model. A directory containing an FCS file.
Model parameters  A comma-separated list of dimensions/parameters/channels that shall be used for the underlying model. Use either parameter names or indexes. All dimensions except for Time will be used if this parameter is not provided.
Model number of recursions  Number of times the FCS training data will be subdivided.
Model dequantize  Option to make all events in the training set unique.
Model sample size  Used to specify the per flowFrame sample size of the data to use in model generation. If NULL, all of the data in x is used. Setting this to a smaller number will speed up processing, at the cost of accuracy.
Output QC screening fingerprint stack  Graphical output in form of "stack" plot.
Output QC screening gate consistency  Graphical output in form of "qc" plot.
Output QC screening plate  Graphical output in form of "plate" plot.
Output csv file  Name for output CSV file.
Output file transformation  Transformation used on data for CSV output file. "raw" is the number of events in each bin, "normalized" is the ratio of the number of events divided by the expected number (the expected number of events is calculated by dividing the total number of events in the instance used to make the fingerprint by the number of features in the fingerprint). "log2norm" is the log2 of the normalized values.
Workflow parameter file  A CSV file containing parameters if module is used in pipeline.

* - required

Output Files

  1. Stack plot - A PNG image showing a stack plot. 

  2. Gates QC plot - A PNG image showing a quality control (QC) plot. 

  3. Plate plot - A PNG image showing a plate plot. 

  4. CSV file - A CSV file, containing one feature vector per row. The columns contain the number of events per bin.
