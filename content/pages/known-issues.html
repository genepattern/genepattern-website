This page includes the known issues for GenePattern running in different browsers, GenePattern running on local servers, and GenePattern module-related issues as of June, 2016 in GenePattern version 3.9.8.

Browsers

See our browser recommendations.

  • Older Java Applet visualizers are no longer supported in Chrome. Please read our blog post for more information.
  • Older Java Applet visualizers are blocked by default in Firefox. Pleaes read this blog post for more information.
  • In Internet Explorer, the Job Status page does not automatically refresh.  Workaround: Refresh the page manually.
  • In Internet Explorer (11, no other versions are supported) you may need to refresh to get new modules to appear in the pipeline designer. This is due to the nature of the caching mechanism in IE.

Java

  • There are known issues regarding some updates to Java 6 and Java 7 that can cause problems for GenePattern users, specifically in using visualizer modules and the file uploader (the Java applet used to upload files larger than 2GB).  For more information, see the GenePattern blog.

Pipelines

  • There is a known issue in GenePattern 3.6 that deleting an attached input file from a pipeline crashes the browser.
  • There is a known issue regarding pipelines created in GenePattern 3.3.3 that have data files included and that have been imported to a new server at version 3.5.0 or higher of GenePattern. If you are having problems with such a pipeline:

1. On the new server, select your pipeline and click edit to open it in the Pipeline Designer.
2. Click Save. You don't need to make any other changes to your pipeline.

Module-Related Issues

  • There is a known issue regarding an update to the Java 6 platform which can cause problems for GenePattern users, specifically in using visualizer modules and the file uploader.  For more information, see this GenePattern blog entry.
  • Some GenePattern modules do not work when the input filename contains a space character. Workaround: Rename the input file without a space character.

 

Local GenePattern Servers

Running a Local GenePattern Server on Mac OS

GenePattern versions 3.9.1 and earlier - this has been resolved in version 3.9.2

  • For many of our users on OS X Mavericks (10.9) and Yosemite (10.10) the older installers will not be allowed to run, due to Apple's security features.
  • There is a known issue with installing on OS X Lion (10.7) or on OS X Mountain Lion (10.8).  If you get an "Installation: Successfulnull" error or an error that looks like this:
1. Move Folder:              Destination: /Library/Frameworks/R.framework/Versions
WARNING - There was a problem copying /Library/Frameworks/R.framework/Versions/2.5

you may need to install R2.5 separately.  Download R2.5 from CRAN and follow the instruction in this section of the Administrators Guide.

  • There are known issues with installing on OS X Mountain Lion (10.8):
    • by default, Mountain Lion does not allow the installation of GenePattern because it is not from a "certified Apple Developer".  Workaround: Set Gatekeeper to allow applications to be installed from any source (see http://www.chriswrites.com/2012/03/5-mountain-lion-features-you-may-have-missed/ for more information).
    • if you select the option to install icons in the dock during install, the icons will not appear in the dock.  They will appear when the server is running.  Workaround: You can place them in the dock manually.

Module-Related Issues

  • The MultiplotPreprocess, CoxRegression, LogisticRegression modules will not run on a local GenePattern server on Mac OS X 10.6, 10.7, or 10.8 or Windows 7 or 8 because R 2.0.1 will not install on these OS X versions.
